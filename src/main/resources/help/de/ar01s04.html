4. Editorfenster

Das Editorfenster zeigt den aktuellen Quelltext des RT-Programms an. Im Editiermodus (Abschnitt 3.1.1: Editiermodus) kann man den Quelltext verändern. Im Simulationsmodus (Abschnitt 3.1.2: Simulationsmodus) zeigt das Fenster die zuletzt ausgeführte Anweisung an. Im Simulationsmodus kann der Quelltext nicht verändert werden.

4.1. Im Editiermodus

Der Editor funktioniert wie jeder beliebige Texteditor unter Windows.

Markieren, Cut, Copy und Paste.  Markieren, Cut, Copy und Paste Text kann wie gewohnt bei gedrückter linker Maustaste markiert werden und dann durch Menü->Bearbeiten->Kopieren oder Strg+C in die Zwischenablage kopiert werden. Analog kann der markierte Text auch ausgeschnitten werden (Strg+X). Text kann auch nur mit der Tastatur markiert werden: Shift gedrückt halten und die Richtungstasten (Pfeile, Bild hoch/runter) zum auswählen der Markierung verwenden. Bei Hinzunahme von Strg wird in Schritten von ganzen Wörtern markiert. Der markierte Text kann dann an beliebiger Stelle im Programm wieder eingefügt werden (Strg+V). Die Markierung kann durch Drücken einer Pfeiltaste aufgehoben werden.

4.2. Im Simulationsmodus

Wird ein RT-Programm simuliert werden die zuletzt ausgeführten Anweisungen farblich markiert.

Tabelle 2. Farbmarkierungen

Farbe Beschreibung
Block Zuletzt ausgeführter Anweisungsblock (Step)
Anweisung Zuletzt ausgeführte Anweisung (Microstep)
Vergleich Ein Vergleich einer if-Anweisung oder switch-case-Anweisung, der wahr ist(Microstep)
Vergleich Ein Vergleich einer if-Anweisung oder switch-case-Anweisung, der falsch ist(Microstep)
Block Anweisungsblock, bei dem die Ausführung abgebrochen wurde (Run)
Block Derzeit ausgewählter Breakpoint
