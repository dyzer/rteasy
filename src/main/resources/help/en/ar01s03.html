3. Main Window

The main window is the "desktop" of RTeasy where you can manage the functions and windows of RTeasy. The menu bar contains the menus:

File (Section 3.1.1.1: Filemenu;)
Edit (Section 3.1.1.2: Edit Menu;)
Simulator (Section 3.1.2.2: Simulator Menu;)
Design (Section 3.1.3.: Design)
Help (Section 3.1.4: Help Menu;)
There a two different modes, the edit mode (Section 3.1.1: Edit Mode) and the simulation mode(Section 3.1.2: Simulation Mode), with different functions activated in the menu bar. Below the menu bar is a bar with different simulator tools (Section 3.1.2.1: Simulator Control). The edit mode disables all buttons in this bar except the button "simulate" that performs a change to the simulation mode.

3.1. Edit/ Simulation Mode

RTeasy has two different states. The edit mode (Section 3.1.1: edit mode) offers functions to work on programs. The simulation mode (Section 3.1.2: Simulation Mode) is only available for completely compilable programs. For further processing you have to switch back to the edit mode.

3.1.1. Edit Mode

This mode is set default after starting an new project or loading an existing project. To switch from the simulation mode back to the edit mode, use the button Edit.

3.1.1.1. File Menu

These functions switch from the simulation mode to the edit mode.

New . closes the current project an creates a new one.

Open(F3) . loads an existing project from your file system.

Save(F2) . saves the current project at a predefined location.

Save As(STRG+F2) . saves the project at the location that is chosen in the appearing dialog box.

Quit . terminates RTeasy.

3.1.1.2. Edit Menu;

These functions are only enabled in the edit mode.

Undo(ctrl+Z) . cancel the last change

Copy(ctrl+C) . copy selected area

Cut(ctrl+X) . cut selected area

Paste(ctrl+V) . insert from clipboard

Pretty Print . tries to compile the program and prints a structured code at success. Otherwise errors and warnings are shown in the message log window (Section 9: Message Log Window).

Settings . opens a dialog box with the current settings.

Language / Sprache: - sets the language of menu titles, buttons, errors and warnings.
Plug & Feel - sets the window design
verbose Warnings - shows all warnings.

3.1.2. Simulation Mode

In the simulation mode you can simulate the predefined and compilable program. You start the simulation mode by using the button Simulate. ( Section 3.1: Edit/Simulation Mode )

3.1.2.1. Simulator Control

The following functions are only available in the simulation mode.

Edit . stops the simulation and switches in the edit mode (Section 3.1.1: Edit Mode)

Reset . sets the simulator to its initial values.

Step . executes the statements in the next cycle.

MicroStep . executes the next statement within a cycle.

Run . executes the program until you stop it or it reaches an end state.

Breakpoints . opens the breakpoint window Section 8: Breakpoint Window to manage breakpoints.

3.1.2.2. Simulator Menu

Reset(CTRL+R) . sets the simulator to its initial values.

Step(CTRL+S) . executes the statements in the next cycle.

MicroStep(CTRL+M) . executes the next statement within a cycle.

Run . executes the program until you stop it or it reaches an end state.

Breakpoints . opens the breakpoint window Section 8: Breakpoint Window to manage breakpoints.

Reset Memories . sets all declared memories to its initial values

Clear Log . clears all window contents. (Section 9: Message Log Window)

3.1.3. Design Menu


3.1.3.1. Show Control Signals

The function "Show Control Signals" generates all control signals of the current program and shows them in the signal window. The Button Close closes the window.


3.1.3.2. Derive Low Level Model

This function derives the low level model of the current program. That implies that all parallel statements will be separated to several single statements.

3.1.3.3. Export to VHDL

This menufunction generates a VHDL-Code from your RTeasy-program. For this it is necessary to check your RTeasy-program for errors. The logwindow will display the errors, that must be repaired. Please notice, that the generated VHDL-Code can have redundant parts and smaller errors. We recommend to look through the generated code and repair and report the errors.

3.1.4. Help Menu

Help . shows the help of RTeasy

About . shows the program information.
